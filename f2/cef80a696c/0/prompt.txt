## Deviations / Decisions

- `launchdPlist` is unexported (lowercase) per Go convention for package-internal pure helpers; the spec said "make it a pure function and unit-test it" without requiring export. If the orchestrator needs it exported for cross-package testing, change to `LaunchdPlist`.
- `StandardOutPath` and `StandardErrorPath` both point to the same log file (stdout+stderr merged) as specified.
- The `launchctl load` fallback is triggered when the `bootstrap` call errors with output containing "already" (already loaded) â€” treated as success â€” OR any other error (falls back to `load`). The spec's "Bootstrap failed" wording is treated as the error message prefix from launchctl, but in practice the fallback fires on any bootstrap error that isn't already-loaded.
- `runCommand` helper is a thin `exec.Command(...).CombinedOutput()` wrapper inlined in daemon.go to avoid polluting lock.go with an import.</result>

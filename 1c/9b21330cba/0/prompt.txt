## Design decisions (where SPEC was ambiguous)

1. **`ScanDir` returns 3 values** `(map[string]Entry, int, error)` â€” the int is the symlink count, printed to stderr by `CmdScan` and ignored by internal callers via `_`.

2. **`buildBase` uses intersection** (paths on both sides after rescan), not union. A path that failed to transfer will have no base entry and thus retry as "new local/remote" on the next run â€” correct eventual-consistency behavior.

3. **Bootstrap is always attempted** when the SSH scan returns exit 127 or "not found" in stderr, regardless of the `--bootstrap` flag (the flag is accepted but treated as a no-op). The SPEC says "Keep it simple: always attempt bootstrap automatically."

4. **`local:` transport copies files directly** (`os.ReadFile`/`os.WriteFile` + `os.Chtimes`) rather than tar, which is faster and sufficient for tests and mounted-volume use.

5. **Conflict annotation** is in `Action.Conflict bool` + `Action.Reason string`; `sync.go` prints `conflict: &lt;path&gt; (local|remote) wins` at apply time. The planner itself produces no side effects.

6. **SSH delete command** uses `$HOME`-expansion for `~/â€¦` paths and single-quote wrapping for absolute paths, consistent with the ScanDir skip-warning for paths containing `'`.
